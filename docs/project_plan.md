# План проекта «ФотоТочка»

## 1. Контекст и цели диплома

- Цель: разработать учебное веб-приложение «ФотоТочка» для покупки и продажи цифровых фотографий (финансовые транзакции имитируются без реальной обработки платежей).
- Основные заинтересованные стороны: студент-разработчик, научный руководитель, потенциальные пользователи (продавцы и покупатели цифровых фото).
- Базовые источники требований: документ `docs/КондратьевДП.docx`, спецификации преподавателя, результаты опросов/интервью (планируется собрать), выдержки из `docs/dp.txt`.

## 2. Стек и архитектура

- Фронтенд: Vue.js 3 + Vite, TypeScript, Vue Router, Pinia для стейта.
- Бэкенд: Django 5 + Django REST Framework; база данных PostgreSQL.
- Инфраструктура: Docker Compose для локальной разработки, GitHub/GitLab CI (определить позже), деплой на Vercel/Render/Heroku (определить по требованиям).
- UI: основная цветовая схема — зелёные и белые оттенки; акцентные цвета уточнить с `UI/UX`.

## 3. Визуальная идентичность и гайд

- Стиль: минимализм с элементами необрутализма (четкие сетки, заметные бордеры, крупная типографика).
- Палитра: `#FFFFFF` фон, `#000000` текст, `#10B981` основной акцент, `#059669` hover, `#ECFDF5` фоновые подложки, `#6B7280` вторичный текст, `#E5E7EB` бордеры, `#EF4444` состояния ошибок.
- Типографика: заголовки — `Playfair Display`, основной текст и UI — `Inter`, fallback системные шрифты.
- Логотип: текстовый знак «ФотоТочка» с точкой-акцентом (см. `docs/dp.txt`), использовать как ориентир для навигации и брендинга.

## 4. Ключевые подсистемы

- Каталог фотографий (листинг, поиск, фильтры, просмотр карточки).
- Симуляция покупок/продаж (оформление заявки, статусы, уведомления без реальной оплаты).
- Пользовательские профили (регистрация, авторизация, роли «покупатель»/«продавец»).
- Управление контентом (загрузка фото, модерация, категории, теги).
- Администрирование (панель модератора, управление пользователями, отчёты).
- Аналитика/логирование (журналы активности, базовые метрики посещаемости).

## 5. Этапы и приоритеты (итерации)

1. **Итерация 0 — Подготовка**
   - Изучить документ `КондратьевДП.docx` и извлечь обязательные требования.
   - Определить структуру репозитория, настроить линтеры и форматирование.
   - Сформировать UI-гайд (палитра, типографика, сетка).
2. **Итерация 1 — Адаптивная главная страница (приоритет)**
   - Создать low-fi/hi-fi макет главной страницы (desktop/tablet/mobile).
   - Реализовать адаптивную страницу на Vue.js с мок-данными (Hero, поиск, новинки, категории, топ авторов, преимущества, CTA).
   - Настроить базовую навигацию, хедер/футер, секции промо и CTA.
3. **Итерация 2 — Каталог и карточка фото**
   - API для списка фото, фильтров, пагинации.
   - UI компонентов каталога, карточка товара, модальное окно просмотра.
4. **Итерация 3 — Пользовательские аккаунты**
   - Регистрация/логин (Django), интеграция на фронте.
   - Личный кабинет продавца с загрузкой фотографий.
5. **Итерация 4 — Симуляция сделок**
   - Оформление «покупки», статусы заказов, уведомления.
   - История операций в профиле пользователя.
6. **Итерация 5 — Администрирование и аналитика**
   - Панель администратора, модерация, отчёты.
   - Метрики, базовая аналитика посещений.
7. **Итерация 6 — Тестирование и защита**
   - Юнит- и интеграционные тесты, e2e (Cypress/Playwright).
   - Подготовка документации, презентации и демонстрации диплома.

## 6. Структура главной страницы (MVP)

- Hero-блок с популярными фото, заголовком «Создавайте. Делитесь. Вдохновляйте.», CTA `Открыть каталог` и вторичным CTA для авторов.
- Поиск и быстрая фильтрация (категория, цвет, лицензия, размер, ориентация).
- Блок «Новинки» — горизонтальный скролл / карусель с lazy-loading.
- Блок категорий: Природа, Животные, Люди, Город, Еда и напитки, Технологии, Абстракции.
- «Топ авторов месяца» — карточки с аватаром, рейтингом, CTA.
- «Преимущества платформы» — три карточки (лицензии, безопасность, удобство).
- Двухколоночный CTA для авторов и покупателей.
- Футер: ссылки на блог, помощь, служебные страницы, контакты.

## 10. Риски и допущения

- Функционал платежей не реализуется — важно явно показывать имитацию.
- Ограниченное время на диплом: чётко контролировать объём фич.
- Нагрузка на UI: требуется адаптивность и доступность (WCAG 2.1 AA), оптимизация загрузки изображений.
- Возможные изменения требований со стороны руководителя — планировать буфер.
